# rabbitloop

Autonomous PR review fixer. Polls GitHub for review comments from bots (CodeRabbit, GitHub Actions) that you've approved with a 👍, dispatches a Claude instance to fix each one, and resolves the thread on success.

## How it works

```
poll GitHub PR
  → find unresolved bot review comments with 👍 from owner
  → for each: fire `claude --dangerously-skip-permissions` with the comment context
  → if Claude echoes a per-invocation nonce, a new commit is detected, and that commit is verified as pushed: resolve the thread via GraphQL
  → sleep and repeat
```

No local state tracking — GitHub's resolved/unresolved thread state is the single source of truth.

## Safety

**rabbitloop is an autonomous code-writing tool. Treat it accordingly.**

- Claude is invoked with `--dangerously-skip-permissions`, so **every tool call (file edits, shell commands, git operations) runs without interactive approval**. This is intentional — the loop needs to run unattended — but it means Claude can change any file in the working tree, run any shell command, and push to the current branch's upstream.
- The prompt is built from **untrusted PR data** (review comment bodies, diff hunks). A malicious or buggy reviewer comment becomes direct input to a permission-skipped Claude session. Treat review comments as you would treat any prompt injection vector.
- Only approved review comments are actioned (👍 from `--owner`), so the trust boundary is: *anyone who can react with 👍 on behalf of the owner can cause arbitrary code changes on the current branch.* Keep that account's GitHub session locked down.
- Only run rabbitloop:
  - on a branch you control (never directly on `main`/`master`),
  - in a checkout with scoped credentials (don't hand it a token with push access to production infra),
  - against PRs from contributors you trust, or review the autogenerated commits before merging.
- Completion is confirmed by a per-invocation random nonce embedded in the prompt — a reviewer quoting a literal "complete" marker cannot spoof completion. A HEAD with no tracking upstream is refused up front, so Claude's pushed commit is always verifiable.

## Usage

```bash
# Dry run — see what would be fixed without invoking Claude
bazel run //devtools/rabbitloop -- --repo arkeros/senku --pr 42 --once --dry-run

# Single pass — fix all approved comments and exit
bazel run //devtools/rabbitloop -- --repo arkeros/senku --pr 42 --once

# Continuous loop — poll every 2 minutes
bazel run //devtools/rabbitloop -- --repo arkeros/senku --pr 42

# Custom interval and owner
bazel run //devtools/rabbitloop -- --repo arkeros/senku --pr 42 --interval 60 --owner arkeros
```

## Options

| Flag | Default | Description |
|------|---------|-------------|
| `--repo` | required | GitHub repository (`OWNER/REPO`) |
| `--pr` | required | Pull request number |
| `--owner` | repo owner | GitHub user whose 👍 triggers fixes |
| `--interval` | 120 | Polling interval in seconds |
| `--once` | false | Run a single iteration then exit |
| `--dry-run` | false | Log what would be done without invoking Claude or resolving threads |

## Prerequisites

- [`gh`](https://cli.github.com/) CLI authenticated with repo access
- [`claude`](https://docs.anthropic.com/en/docs/claude-code) CLI installed

## Supported bots

- [CodeRabbit](https://coderabbit.ai/) (`coderabbitai`)
- GitHub Actions (`github-actions[bot]`)
