# workstation

A debian13-based developer-VM OCI image, published as
`ghcr.io/arkeros/senku/workstation`. Designed primarily for
[exe.dev](https://exe.dev/) but is plain Linux + systemd ‚Äî no exe.dev-specific
runtime requirements baked in.

## Contract

**What's in:**

- Debian 13 (trixie) base
- systemd as PID 1 (via `init-wrapper`)
- `exedev` user (UID 1000), passwordless sudo, lingering enabled
- Tier-0/Tier-1 apt set:
  `systemd systemd-sysv sudo ca-certificates dbus-user-session curl jq git direnv fish`
- Out-of-band binaries: `bazelisk`, `claude` (verified by sha256)
- exe.dev plumbing: `shelley.{service,socket}`, `exe-setup.service`,
  `/usr/local/bin/init`
- `LABEL "exe.dev/login-user"="exedev"`

**What's deliberately *not* in** (vs. upstream `exeuntu`):

- The Ubuntu meta-packages: `ubuntu-server`, `ubuntu-standard`, `ubuntu-dev-tools`
- Headless Chrome stack (`libnss3`, `libgtk-3-0`, fonts, `chromedp/headless-shell`
  multi-stage)
- Docker-in-Docker (`docker.io`, `docker-buildx`, `docker-compose-v2`)
- Tailscale, mitmproxy, imagemagick, ffmpeg, system monitors (`atop`/`btop`/`iotop`/`ncdu`)
- Go, uv, codex, pi (re-add per-user via setup script if needed)
- man pages (no `unminimize`)

If you need any of those, install via exe.dev `new --setup-script` or
extend this image in a sibling target.

## Naming

The user-inside-the-image stays `exedev` for wire-protocol parity with exe.dev's
`shelley` (the SSH-replacement service expects exactly the env/paths upstream
uses). The artifact and source-tree names (`workstation`,
`devtools/workstation/`) decouple from the consumer.

## Versioning & publish

- Tagged `:latest` on every push
- Plus content-addressed `:sha-<short-git-sha>` (TODO)
- No floating `:stable` (the user gates rollouts via exe.dev's
  `defaults write dev.exe new.image`)

## Usage

```bash
# one-off
ssh exe.dev new --image=ghcr.io/arkeros/senku/workstation:latest

# default for all new VMs
echo ghcr.io/arkeros/senku/workstation:latest | \
  ssh exe.dev defaults write dev.exe new.image
```

## Scope creep watch

This package will accumulate "developer environment" concerns over time. That's
fine ‚Äî the directory is the right anchor for them ‚Äî but resist adding artifacts
that aren't *for* the developer's interactive VM. Production runtime images live
under `//oci/distroless/`.
