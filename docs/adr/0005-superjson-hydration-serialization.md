# superjson for hydration data serialization

Data returned from `preload` is serialized to the HTML data island via superjson rather than plain `JSON.stringify` or devalue, so route preloads can return `Date`, `Map`, `Set`, `BigInt`, and similar non-JSON-safe values without manual round-trip code at every preload/component boundary (and without silent bugs from `Date`-becomes-string asymmetry). superjson was picked over devalue because its output is JSON-shaped — the wire format composes natively with the `<script type="application/json">` data island that env and locale also use, and hydration parses with a single `JSON.parse` + `superjson.parse` chain that doesn't require the alternative-script-tag-escaping that devalue's expression-shaped output would force; the cost is ~7KB additional bundle weight (vs ~3KB for devalue) and a small wire-format overhead from superjson's type-metadata sidecar.
