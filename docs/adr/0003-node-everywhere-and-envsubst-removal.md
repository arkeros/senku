# Node-everywhere prod runtime, envsubst removed

The `react_ssr_app` prod OCI image is a single distroless-Node process serving both SSR-rendered HTML and static assets (bundles, CSS, hashed assets, runtime config); nginx is gone from the image, and the asset/SSR split is a routing concern inside one Hono process rather than a two-binary topology. Runtime config is read directly from `process.env` at server boot and inlined into each HTML response as a `<script type="application/json">` data island, replacing the previous envsubst-at-container-start mechanism — operators set environment variables on the container the same way every other Node app does, there's only one source of truth for env values, and the `_env_tpl` / `_env_dev` Bazel targets and `default.conf.tpl` envsubst integration are deleted as part of the migration. The single-process design is intentional ("what you build is what you ship" — same Hono server runs in dev devserver and prod image, just configured differently); reverse-proxy/CDN concerns belong at the deployment topology layer (Cloud Run, k8s ingress), not inside the image.
