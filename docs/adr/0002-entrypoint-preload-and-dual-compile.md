# Entrypoint-shaped preload + same-file named-export dual-compile

Per-route data is declared as a `preload` named export on the route component file (alongside a `meta` export); the framework awaits it server-side and injects the result into the component as **props**, deliberately *not* via `useLoaderData()`. This mirrors Relay's Entrypoint pattern (preload-in-parallel-with-lazy + props-not-hooks), so the future switch to GraphQL+Relay is additive — preload's plain-JSON return becomes a `PreloadedQueryReference`, the prop wiring stays — rather than a rewrite of the data layer. To keep colocation safe, a custom Babel transform produces dual outputs from each route file: `.client.js` (with `preload` and `meta` exports stripped and dead imports swept) and `.server.js` (full module); two Bazel actions run in parallel, two cache entries, no tree-shaking trust required.
